## Detailed tooling stack

The high-level stack lives in CLAUDE.md. The full per-milestone
tooling reference is here.

### Vector DB (Milestone 5)
Qdrant or pgvector. Decision deferred to M5 Phase A — pgvector if we
already have Postgres in the stack for tenant metadata; Qdrant if we
want pure vector ops at scale.

### Retrieval (Milestone 5)
- **BM25 + dense embeddings** with **RRF fusion at k=60**
- **Cross-encoder reranker:** `bge-reranker-v2` local /
  Cohere Rerank 3.5 cloud (ADR comparing the two)
- **Glean-style ACL filtering** — post-retrieval, pre-context-
  assembly. Never trust the LLM with unfiltered context.
- **Multi-tenant isolation pattern:** Silo per enterprise tenant,
  Pool for standard / free tier (ADR)

### Observability
- **OpenTelemetry GenAI semconv** (experimental, dual-emit via
  `OTEL_SEMCONV_STABILITY_OPT_IN`) instrumented via **OpenLLMetry**
  (Traceloop)
- Backend: **Langfuse** (default) — ADR comparing Langfuse / Arize
  Phoenix / Helicone / Braintrust
- **Phoenix Span Replay** for deterministic replay of non-
  deterministic reasoning + tool-call failures (M8)
- Prometheus + `structlog` for non-LLM signals
- Wired from M1 (basic spans) — full coverage at M8

### Agents (Milestone 6)
- **LangGraph** for state / flow + **Pydantic AI** for typed tool
  I/O inside nodes
- ADR justifying choice over OpenAI Agents SDK / smolagents /
  Strands / Microsoft Agent Framework
- **Anthropic prompt caching** for stable system prompt + tool
  definitions: 5-min cache write 1.25x cost (pays off after 1 read),
  1-hour write 2x (pays off after 2 reads), 0.1x read cost.
  Workspace-isolated since Feb 2026.

### Structured outputs (Milestone 7)
- **xgrammar** (vLLM v1 default, pushdown-automaton-based, ~100x
  faster than Outlines via vocab partitioning)
- ADR comparing xgrammar / Outlines / lm-format-enforcer / BAML /
  Instructor
- OWASP LLM Top 10 2026 + OWASP Agentic Top 10 2026 mapped to
  red-team test cases

### Evals (Milestone 9)
Three-layer split (ADR justifying):
- **Inspect AI** (UK AISI) — capability / safety evals at model
  layer
- **DeepEval** — pytest-style application-layer
- **Ragas** — retrieval-specific
- Reasoning-model-as-judge with binary pass/fail rubric (Hamel
  Husain doctrine — see Cross-cutting conventions below)

### Cost / routing (Milestone 10)
- **RouteLLM**-style router for tier selection
- **Portkey**-style gateway for policy / fallback / cache
- **OpenRouter** as marketplace reference
- ADR on the router-vs-gateway distinction

### Deployment (Milestone 11)
- **KServe LLMInferenceService CRD + llm-d** backend (both CNCF —
  KServe incubating Nov 2025, llm-d sandbox March 2026). Don't
  reinvent; deploy onto the converged ecosystem.
- **Knative** for request/concurrency-based autoscaling with
  scale-to-zero (the cost-control primitive)
- **LitmusChaos** for the chaos suite (CNCF incubating; ships an MCP
  server — drive chaos from the agent loop = high-leverage demo)

### Diagrams and documentation
- **Mermaid** in markdown for sequence / flow / component diagrams
  in milestone reports and ADRs (renders natively on GitHub — no
  separate build step). Use from M1.
- [**KubeDiagrams**](https://github.com/philippemerle/KubeDiagrams)
  for auto-generated K8s architecture diagrams from manifests, Helm
  charts, and live cluster state. Used from M11; opportunistically
  earlier via the `kubectl diagrams` plugin. SVGs land in
  `docs/diagrams/`, regenerated by CI.
- [**Structurizr**](https://structurizr.com/) (C4 model) for
  high-level architecture documentation in `docs/architecture.md`
  (M11+).
- [**DeepWiki by Cognition**](https://deepwiki.com) for
  auto-generated repo wiki. **Submission deferred to Milestone 12**
  (end of build) so the wiki captures substantive end state, not
  scaffolding. Add the badge to README the day M12 launches:
  `[![Ask DeepWiki](https://deepwiki.com/badge.svg)](https://deepwiki.com/sriteja28/production-llm-platform)`.

## Cross-cutting conventions (apply to every milestone)

1. **Cost-per-correct-answer in every milestone report.** Not just
   M12. This is the metric that distinguishes the project from
   generic LLM serving portfolios at the $400K hire tier.
2. **Hamel Husain eval doctrine.** Every milestone with quality
   claims uses binary pass/fail judge + classifier validation
   (precision / recall on a labeled set of 100+ examples) + paired
   bootstrap or McNemar's test for statistical significance — not
   t-test on Likert means.
3. **OpenTelemetry GenAI semconv from M1.** All LLM instrumentation
   goes through OpenLLMetry against the GenAI semconv from the first
   milestone. M8 brings full agent-loop coverage and Phoenix Span
   Replay; M1-M7 wire basic spans against the same schema.
4. **OWASP coverage doc.** `docs/owasp-coverage.md` maps OWASP LLM
   Top 10 2026 + OWASP Agentic Top 10 2026 to specific milestone
   test cases. Updated as milestones complete.
5. **vLLM disagg prefill caveat.** Disagg prefill is still
   experimental as of April 2026; improves tail ITL but not
   throughput. Frame as a tail-latency vs throughput tradeoff with
   Sarathi-Serve chunked prefill as the alternative — never claim
   throughput win.
